唐后世界的模型，
说出它的惊奇。

两个小实验：用“惊奇度”测量语言断裂；用“古文续写器”观察唐前语感如何面对陌生概念。

这不是预言机器。它是一面有年代边界的镜子。
实验边界01 惊奇度02 续写器实施路线解释纪律
FIRST, A CLEAR CLAIM

我们要测量的，不是未来；
而是唐前语言世界的边界。

当前模型只见过唐代及以前的古籍。它学的是“前文出现以后，下一个字符通常是什么”的规律。因此，它可作为一个受限的语言实验对象：当它遇到唐后表达或现代概念时，究竟有多不习惯？它会用哪些唐前句式去续接？

请把结论说窄。 高惊奇度并不证明某件事“历史上不可预测”，更不证明模型理解历史因果。它只表示：对于这个唐前语料训练出的字符模型，这段表达在语言形式和字符搭配上不常见。
唐前语料经、史、子、集、医学、算书与唐诗。模型只从这里学习字与字的关联。
→
字符级 LSTM读入前文，给每个候选字符分配下一个字的概率。
→
两种观察一是“这段文字有多意外”；二是“从此开头会续出什么”。
EXPERIMENT 01 / SURPRISE MAP

唐前惊奇度实验

问题

它觉得哪一种表达最陌生？

将长度相近的文本片段逐字送入已经训练好的 best.pt。每当模型实际看到一个字，就记录它之前给该字的概率。

若模型原本认为“下一个字最可能是之、曰、者”，实际却出现了一个它极少在此语境见到的字，惊奇度就升高。

产物

一张“语言断裂地图”

把片段按时代或主题分组，比较平均惊奇度：唐前、宋元、明清、近现代；或制度、科技、战争、日常生活等主题。

图上的峰值是“模型语言预期最失效”的地方，值得回到原文细读。

惊奇度（每字符） = 平均值 [ − log P（实际出现的下一个字 ｜ 前文） ]

低：模型常能预料这个字。　高：这个字或这种组合令模型意外。
只在同一模型、同一清洗规则、相近长度的文本之间比较。

实验步骤

准备对照文本
每组至少 20 段，尽量每段 150–300 字。保留来源、年代、体裁和主题。
统一清洗规则
全部转成 UTF-8；压缩连续空白；不要悄悄删掉“不像古文”的字。
报告覆盖率
当前模型没有 <unk>（未知字）机制。测试字不在字符表中时，必须统计并单列，不能假装模型评过它。
计算并分组
记录平均惊奇度、困惑度、可评分字符比例；再按时代/主题求均值，而非只挑一个耸动例子。
你可以比较什么好的设计不能得出的结论
时代同体裁短文：唐、宋、明、近代各 20 段“模型证明宋代不可预测”
概念用相近文体比较交通、政制、通信的描述“高分概念是人类绝对无法想象的”
文本风格史书、诗歌、论说文各自内部比较把诗歌与白话新闻的分数直接等同
EXPERIMENT 02 / CONTINUATION DESK

古文续写器

它的工作很简单：给出开头，LSTM 逐字预测后文。它不会“查资料”或“理解真相”，但能暴露它从唐前语料吸收的常见句式、官职词、史书节奏与诗文搭配。

子曰观察：伦理、修身类搭配
臣闻天下之势观察：论说与政论节奏
若有飞车日行千里观察：古典语感如何包裹陌生物

生成命令：先用最佳模型

python sample.py checkpoints/full/best.pt \
  --prompt "臣闻天下之势" \
  --length 240 \
  --temperature 0.7 \
  --top-k 30 \
  --device cuda
参数作用本实验建议
prompt让模型先读的开头每次只改开头，方便比较
length续写的字符数150–300，太长更容易跑题
temperature随机性；越高越冒险0.6–0.8
top-k只从最可能的 k 个字中抽取20–40，减少罕见乱码

生成结果应标注“模型续写，非古籍原文”。 不要把它当作史料引用、古文翻译、古人观点或可靠答案。把它当作一个有趣的语料镜像。

IMPLEMENTATION / AFTER EPOCH 10

从已训练模型到实验的实施路线

训练尚未结束时不要动 checkpoints/full/。等监控显示进程结束后，确认 best.pt 存在，再开始实验。续写器可以立即运行；惊奇度评分器需要新增一个只读 checkpoint 的小脚本。

锁定实验版本
复制并记录 best.pt 的文件名、训练参数、语料版本与最终验证损失。
先做三组小样本
唐前、唐后、现代各 20 段；先验证评分器与字符覆盖率，再扩大数据集。
实现评分器
读取 checkpoint、编码字符、逐字取实际目标字的 log probability，输出 CSV。
审阅与展示
画分布图、列出高惊奇片段、人工检查上下文，再写出有边界的结论。

将来评分器的命令目标

# 这个脚本将在训练结束后新增
python score_surprise.py \
  --checkpoint checkpoints/full/best.pt \
  --input experiments/modern_transport.txt \
  --output results/modern_transport.csv \
  --device cuda

输出应至少包含：片段 ID、原始字符数、可评分字符数、未知字符数、平均负对数概率、困惑度、最高惊奇位置。这样别人才可以复查你的结论。

THE DISCIPLINE OF INTERPRETATION

一条好的结论，应该长这样。

“在本实验使用的唐前语料、字符级 LSTM 和统一清洗条件下，某类唐后文本的平均字符惊奇度较高。这说明该模型的语言预期对这类表达适配较差；它不构成对历史必然性、人物思想或技术可预测性的证明。”

这条限制不是扫兴，恰恰让实验变得有意思：我们不是把模型神化成先知，而是用它具体地看见一种语料传统的惯性、盲区与想象力。

第一个问题不该是“它预言了什么？”
而应是：“它在哪个字开始感到陌生？”
