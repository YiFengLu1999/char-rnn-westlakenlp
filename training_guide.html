教孩子接龙你指着“床前明月”，让孩子猜下一个字。猜“色”，你说错；答案是“光”。孩子稍微调整记忆。几百万次以后，他开始知道“明月”附近常出现什么。

这个项目做的事情就是：每看见一个字符，预测紧随其后的字符。模型从不直接背诵“唐诗规则”，规则是它在无数次猜测和纠错中自己压缩出来的。

一句话目标：让正确的下一个字符获得更高概率，让错误字符获得更低概率。

先记住：RNN 是一个家族。普通 RNN、GRU、LSTM 都是“带记忆的逐步阅读器”。项目默认用 LSTM，因为它比普通 RNN 更不容易忘掉较远的上下文；你仍可用 --model rnn 训练普通 RNN。

03

第一关：文字必须先变成数字

电脑不会直接理解“月”。代码先建立一张字典，让每个 Unicode 字符拥有唯一编号。

图书馆借书证图书馆不靠“那本蓝色的书”管理藏书，而给每本书一个编号。字符表也一样：可能把“月”编成 8,421，把“明”编成 7,193。编号本身没有诗意，只为方便计算。

CharacterTokenizer.from_text 用 set(text) 找出所有不同字符，再排序以保证每次编号一致。当前语料约有 17,507 个不同字符，包括汉字、标点和换行。

char_rnn/data.py｜第 18–38 行
@classmethod
def from_text(cls, text):
    return cls(tuple(sorted(set(text))))

def encode(self, text):
    mapping = self.stoi
    return torch.tensor(
        [mapping[character] for character in text],
        dtype=torch.long,
    )

Embedding 是什么？编号只是门牌号，不能表示意义。模型第一个可学习层 nn.Embedding 会把每个编号换成一串可训练的小数。可以把它想成给每个字发一张“性格卡”：训练后，“山”“水”“云”的卡片会形成对预测有用的模式。

为什么这是字符级，而不是词级？
代码逐个迭代 Python 字符串，因此“明月”会得到“明”和“月”两个 token，不需要中文分词。优点是不会遇到生词切分问题，也能学习古汉语单字组合；代价是序列更长，模型需要更多步才能理解长距离关系。
为什么生成提示里不能出现训练集没有的字符？
模型的输出口只有字符表里的 17,507 个位置。没见过的字符没有编号、Embedding 和输出概率，因此 encode 会明确报错，而不是悄悄替换。

第二关：把课本裁成练习题

输入和答案只错开一个字符。模型看到前一个字，目标永远是下一个字。

INTERACTIVE / 下一字练习

逐步看看模型在做哪一道题

透明尺遮住答案老师拿透明尺盖住每行后半截，让学生一次只猜下一个字。序列长度 seq_length=128，就是每张练习卷连续放 128 道接龙题。

假设原文是“明月松间照”：输入为“明月松间”，答案为“月松间照”。整排向左错开一格，就自动产生了四道监督学习题。

char_rnn/data.py｜第 66–75 行
inputs  = tokens[:usable].view(batch_size, -1)
targets = tokens[1:usable + 1].view(batch_size, -1)

self._batches = [
    (inputs[:, start:start + seq_length],
     targets[:, start:start + seq_length])
    for start in range(0, inputs.size(1), seq_length)
]

batch_size 是同时批改多少张练习卷。更大通常能更充分利用 GPU，但会占更多内存；它不是“读多少字以后才学习”。每一个 batch 都会进行一次参数更新。

train_fraction=0.95 把大多数练习题用来学习，留下约 5% 当从未参与改答案的模拟考试。

05

第三关：RNN 带着笔记往前读

普通网络每次只看眼前；RNN 多带了一张会更新的便签，这张便签就是 hidden state。

驿站传递密札每个字符是一座驿站。上一站把一张密札交给下一站，密札记录“目前读到什么语境”。下一站同时看新字和密札，再改写密札继续传递。

hidden state 不是原文副本，而是一组压缩数字。它可能编码句式、主题、韵律等对下一字有用的信息。LSTM 额外使用门控机制，像决定哪些旧笔记该保留、哪些该擦掉。

char_rnn/model.py｜模型结构与 forward
self.embedding = nn.Embedding(vocab_size, embedding_dim)
recurrent_class = {
    "rnn": nn.RNN,
    "gru": nn.GRU,
    "lstm": nn.LSTM,
}[model_type]
self.recurrent = recurrent_class(embedding_dim, hidden_size, ...)
self.output = nn.Linear(hidden_size, vocab_size)

def forward(self, tokens, hidden=None):
    embedded = self.embedding(tokens)
    activations, hidden = self.recurrent(embedded, hidden)
    return self.output(activations), hidden
embedding_dim=128 是什么？
每个字符的“性格卡”有 128 个可学习数字。更大能容纳更丰富的表示，也会增加参数和计算量。它与 hidden_size 不必相同。
hidden_size=256 是什么？
驿站密札有 256 个数字。越大，记忆容量越高，但训练更慢、占用更多显存，也更容易过拟合。
num_layers=2 是什么？
不是一位学徒，而是两位叠在一起：第一层把读到的模式交给第二层，第二层形成更抽象的上下文。层数超过 1 时，dropout 会在层之间随机遮住部分信号，防止死记硬背。
06

第四关：猜错以后，怎样真正学会

模型第一次几乎乱猜。学习发生在“算错多少 → 追查责任 → 微调所有旋钮”这三步。

万旋钮的调音台模型里有数百万个小旋钮。loss 是刺耳程度；反向传播沿电路倒查每个旋钮对噪声负了多少责任；AdamW 优化器再把每个旋钮轻轻拧向更悦耳的方向。

CrossEntropyLoss 会惩罚模型没有把高概率给正确字符。loss.backward() 计算梯度；optimizer.step() 才真正修改参数。

train.py｜第 79–90 行：一次 epoch
for inputs, targets in batches.split("train"):
    optimizer.zero_grad(set_to_none=True) # 擦掉上题草稿
    logits, _ = model(inputs.to(device))       # 作答
    loss = criterion(logits.reshape(-1, vocab_size),
                     targets.to(device).reshape(-1))
    loss.backward()                            # 追查责任
    nn.utils.clip_grad_norm_(model.parameters(), grad_clip)
    optimizer.step()                           # 调整旋钮

epoch 就是把全部训练练习题完整做一遍。10 epochs = 全书反复练 10 遍。它不是越多越好：训练分数一直进步、模拟考试却退步时，就是开始背答案而没有更好地掌握规律。

grad_clip=1.0 像给方向盘加限位器。RNN 的梯度偶尔会突然极大，裁剪可以避免一次过猛调整把已经学到的东西全部撞坏。

07

第五关：考试不参与改答案

训练 loss 代表练习册表现，validation loss 代表闭卷考试。真正值得保存的是考试成绩最好的一次。

练习册与密封试卷如果学生背下练习册，练习分数会很高，却不一定会做新题。验证集像老师提前密封的试卷：训练时绝不能根据它修改答案。

evaluate 使用 torch.no_grad()，只统计成绩，不计算梯度。每轮结束都会保存 last.pt；只有验证 loss 创新低时才覆盖 best.pt。

train.py｜第 91–100 行
val_loss = evaluate(model, batches.split("val"), criterion, device)
save_checkpoint(output_dir / "last.pt", model, tokenizer, ...)

if val_loss < best_loss:
    best_loss = val_loss
    save_checkpoint(output_dir / "best.pt", model, tokenizer, ...)

一份 checkpoint 里面装了什么？

char_rnn/checkpoints.py 把四类东西装进同一个文件：模型结构配置、已经学到的参数、字符表，以及训练到第几轮和 loss 历史。它像“学徒大脑 + 字典 + 成绩册”的完整冷冻存档。

看 loss 的方向，不迷信绝对值：训练 loss 和验证 loss 一起下降通常是好事；训练下降而验证连续上升通常是过拟合；出现 nan 往往意味着数值不稳定，应减小 learning rate。
08

毕业表演：让模型逐字续写

生成不是一次写出整首诗。模型每次只抽一个字，把这个字接回输入，再猜下一个。

摸球写诗模型把 17,507 个字符放进抽奖箱，每个字符的球数由概率决定。“月”的球很多就更容易抽到，但不保证每次相同。抽到一个字后，再据此重新装下一箱球。

temperature 调整抽奖箱的冒险程度；top-k 只保留最有希望的若干候选；greedy 则永远选概率最高者。

终端｜使用最佳存档生成 300 字
python sample.py checkpoints/best.pt \
  --prompt "明月" \
  --length 300 \
  --temperature 0.8 \
  --top-k 30

09

遇到报错时，像侦探一样排查

错误不是“我不适合训练模型”，只是流水线上某个环节没有接好。先看最后一行报错，再按症状定位。

ModuleNotFoundError: No module named 'torch'
依赖没有装进当前环境，或终端没有激活虚拟环境。运行 source .venv/bin/activate，确认命令行前出现 (.venv)，再运行 pip install -r requirements.txt。
CUDA out of memory / MPS 内存不足
先把 --batch-size 16 改成 8 或 4；仍不足再把 --hidden-size 256 改成 128。batch size 是最直接的显存旋钮。
训练非常慢，看起来像卡住
完整语料约 2,008 万字符，一轮包含很多 batch。先确认终端是否持续输出、活动监视器是否有 CPU/GPU 使用率。第一次可用 --epochs 1 --hidden-size 64 --num-layers 1 --batch-size 8 做通路测试。
生成时提示字符不在 vocabulary
提示词含有训练语料从未出现的字符。换一个常见古汉语提示，或把该字符加入训练数据后重新训练；旧 checkpoint 的字符表不会自动扩大。
生成结果一直重复
不要用 --greedy；尝试 --temperature 0.8 --top-k 30。若仍严重重复，可能训练不足、过拟合或普通 RNN 的记忆能力有限，可改用 LSTM。
loss 变成 nan
把 --learning-rate 3e-3 降到 1e-3 或 3e-4，保持梯度裁剪开启，并检查数据中是否存在异常。

第一次完整训练检查单

你真正需要记住的五句话

  1. 语料是课本，字符表是编号册。
  2. 输入和答案错开一字，模型永远在猜下一字。
  3. hidden state 是读到当前位置时留下的压缩笔记。
  4. loss 衡量错得多严重，反向传播负责追查每个参数的责任。
  5. 训练集用来学习，验证集用来选出真正泛化更好的 best.pt。
